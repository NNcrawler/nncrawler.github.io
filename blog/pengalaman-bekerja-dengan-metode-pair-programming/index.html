Keuntungan Bekerja Dengan Metode Pair Programming

programming

March 13, 2021

Pair programming adalah metode penulisan software dengan menempatkan 2 programmer pada tugas yang sama. 2 perusahaan terkenal Indonesia yang menggunakan pair programming adalah KMK dan Gojek (we are hiring). Pair programming diperkenalkan sebagai salah satu teknik dalam extreme programming(XP). Rasionalisasi dari pair programming pada XP adalah “Code review itu efektif. Kenapa tidak review secara terus menerus saja?“. Selain code review secara berkelanjutan, ada juga keuntungan-keuntungan lain dari pair programming.

Knowledge Transfer

Ketika menulis software, programmer harus memiliki kompetensi yang dibutuhkan. Komepetensi ini dapat berupa bahasa pemrogramman, tools, library, dan lain sebagainya. Ketika seseorang ditempatkan pada projek baru, belum tentu semua kompetensi sudah dimiliki. Pair programming dapat mempercepat proses peningkatan kompetensi tersebut. Orang baru tersebut dapat menjadi pair pasif dan mengajukan banyak pertanyaan kepada pasangannya.

Ketika saya baru memulai karir sebagai software engineer, saya mengerjakan migrasi deployment tool dari internal tool ke Helm. Saya mendapatkan pair yang sangat ahli dalam menggunakan tools untuk meningkatkan produktivitas. Dia sangat cepat sampai sampai saya sulit mengikuti pergerakannya. Bayangkan saja, setup yang waktu itu dipakai adalah 2 monitor. 1 monitor dipasang secara vertikal yang berfungsi menampilkan terminal. Biasanya terminal pada monitor itu dibagi 4. Monitor satunya digunakan untuk menampilkan code editor dan internet browser. Dia tidak menggunakan mouse sama sekali dan pergerakannya sangat cepat. Dari pengalaman itu, saya banyak mengadopsi tools yang dia gunakan. Pair saya pun membantu menginstall beberapa tools pada laptop saya. Jika tanpa pair, mungkin dibutuhkan waktu berbulan-bulan untuk dapat mengadopsi tools-tools tersebut.

Menghindari Bug

Menghindari bug adalah keuntungan yang terpikirkan secara langsung berhubungan dengan code review. Dengan pair programming, satu orang dapat berperan sebagai executor, seorang lagi dapat berperan sebagai reviewer.

Pada pengalaman saya banyak bug atau kesalahan fatal yang terhindarkan karena pair programming. Mulai dari keselahan ringan seperti typo, sampai kesalahan fatal seperti menghapus deployment secara tidak sengaja.

Mendapatkan Implementasi Optimal

Ketika menulis software seorang diri, implementasi hanya sebatas pada pemahaman seorang programmer. Dengan pair programming, akan ada seorang programmer lagi yang dapat memberi masukan berdasarkan pengalamanya. Yang menarik adalah dampak ini tidak seperti penjumlahan dimana pengetahuan programmer A + programmer B. Dalam kasus terbaik, dampaknya adalah perkalian. Ini dikarenakan sebagai pair, programmer dapat menantang ide satu sama lain dan melahirkan ide yang tidak pernah terpikirkan sebelumnya. Kombinasi ini dapat menghasilkan software dengan kualitas tinggi.

Implementasi software terbaik yang pernah saya tulis pun tidak terlepas dari pair programming. Pada saat itu saya dan pair saya sering berbeda pendapat. Kami berdua saling menunjukan kelemahan ide satu sama lain. Tidak jarang kami terpikir ide yang tidak terpikirkan sebelumnya karena diskusi-diskusi tersebut.

Istirahat Sejenak

Ada saat dimana sesorang lelah dan tidak semangat bekerja. Pada saat itu, programmer dapat berperan sebagai pair pasif dan beristirahat sejenak. Jika dilakukan dengan benar, bahkan produktifitas bisa jadi tidak terputus dengan bergantian sebagai pair aktif dan pasif.

Mengurangi Truck Factor

Bayangkan sebuah software project yang terdiri dari 3 orang. Setiap orang mengerjakan bagian masing masing. Pada suatu hari, salah seorang programmer tertabak truck dan tidak dapat melanjutkan project (mungkin si programmer pindah ke dunia lain). Jika project sudah berjalan cukup lama dan sudah banyak yang dikerjakan, untuk dapat melanjutkan pekerjaan programmer yang tertabrak dibutuhkan waktu yang tidak sedikit. 2 orang lainnya harus memahami apa yang sebelumnya dikerjakan dan contextnya. Dengan pair programming, programmer yang menjadi pasanganya akan dengan mudah melanjutkan pekerjaan yang ditinggalkan.

Pada dunia nyata mungkin kendala yang paling sering terjadi bukanlah tertabrak truck, melainkan sakit. Ketika seseorang dalam tim tersebut sakit dan timeline dari project tidak dapat diundur. Maka, programmer lainnya dapat menggantikan secara langsung. Oleh karena keuntungan ini juga Gojek dapat menerapkan unlimited sick leave policy 👍.

Kesimpulan

Dari berbagai keuntungan diatas pair programming dapat menjadi pilihan penulisan software. Metode pair programming tidak terlepas dari kekurangan. Kekurangan dari pair programming akan diuraikan dalam blog lainnya. Ada pula topik lainnya seperti setup pair programming yang dapat diterapkan.
